# LP_practice

このリポジトリは、LP制作を段階確認型で進めるための練習・運用用リポジトリです。

## 目的

商品・サービスのLP制作を、以下の流れで安全に進めるために使います。

1. 競合調査
2. 類似商品・代替手段調査
3. ターゲット選定
4. 訴求軸の提案
5. LP構成案作成
6. LP本文作成
7. HTML/CSS化
8. GitHub反映
9. Pull Request作成
10. 確認後にマージ

各ステップでは、ユーザーが「OK」と言うまで次に進まない運用にします。

## ファイル構成

### LP_AGENT_INSTRUCTIONS.md

段階確認型LP制作エージェントとして振る舞うための指示書です。

LP制作を始めるときは、この内容を前提に、AIへ以下のように依頼します。

```text
LP_AGENT_INSTRUCTIONS.md の内容に沿って、段階確認型LP制作エージェントとして進めてください。

最終的にはLPを作成したいです。
ただし、一気に最後まで進めず、各ステップごとに止めてください。
私が「OK」と言ったら次に進んでください。

まずはステップ1：事前ヒアリングから始めてください。
```

### LP_PROJECT_SUMMARY_TEMPLATE.md

LP制作が一通り終わった後に、今回のプロジェクト内容を要約するためのテンプレートです。

最初に埋める必要はありません。
競合調査、ターゲット選定、訴求軸、LP本文、実装まで終わった後に整理します。

## 運用ルール

- 最初からテンプレートを埋めようとしない。
- 商品・サービス概要だけでも開始できる。
- ターゲットが未定の場合は、リサーチ後に選定する。
- LP本文やHTML/CSSは、ターゲットと訴求軸が確定してから作る。
- GitHubで作業するときは、mainに直接変更しない。
- 作業用ブランチを作り、Pull Requestで確認する。
- マージはユーザーが明示的にOKした後に行う。

## 注意

保険・金融・医療・投資など、表現に注意が必要な商材では、断定表現や過度な不安訴求を避けます。
最終的な公開前には、人間が必ず内容を確認します。
